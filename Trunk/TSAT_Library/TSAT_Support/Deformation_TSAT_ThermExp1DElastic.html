TSAT: Thermal Expansion 1D Elastic Library Block

This function calculates the length or radius of a structure that has expanded or contracted due to thermal expansion. The block accepts the previous/reference length (Lo), previous/reference temperature (To), and current temperature (Tf) as its inputs. The user has the option to specify a constant thermal expansion coefficient or to incorporate temperature dependence by providing data that correlates the thermal expansion coefficient with temperature. The bond length of a specimen has the following relation when considering temperature dependence.

$$ L = L_0 \left[ 1 + \int_{T_0}^{T_f} \alpha (T) dT \right] $$

where $$ T = temperature $$ $$ T_0 = previous/reference \: temperature $$ $$ L_0 = previous/reference \: length \: at \: temperature \: T_0 $$ $$ T_f = current/final \: temperature $$ $$ L = current/final \: length \: at \: temperature \: T_f $$ $$ \alpha = thermal \: expansion \: coefficient $$

In the case that the thermal expansion coefficient is constant the final length can simply be expressed as:

$$ L = L_0 \left[ 1 + \alpha \left( T_f - T_0 \right) \right] $$

This block provides the option to evaluate the first expression in the case where the thermal expansion coefficient is dependent on the temperature. This is done given a temperature and corresponding thermal expansion array which are used to integrate over the appropriate temperature range. The other option is to specify a constant thermal expansion coefficient which is used to evaluate the second expression.

This function could be used for steady-state calculations. Theoretically it can also be used in dynamics simulations where To and Lo are the temperature and length from the previous time-step, Tf is the current temperature, and L is the current length.

Note that if the temperature dependent option is being utilized then the temperature array, T_data_M, should be monotonically increasing and have at least 2 elements. The thermal expansion coefficient array, alpha_data_M, should be the same size as T_data_M. Generally all units must be specified consistently. For example, if temperature is in Celsius all temperature units must in Celsius. Below is a table that lists the proper units for each input and parameter in standard English and SI units. For definitions of each of the inputs and parameters (masked variables) refer to the tables near the bottom of the page.
